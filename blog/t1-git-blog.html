i Git knocked down

(but i get up again)

july 10, 2015


I'll be honest, last week i had never even heard of Git. And now here I am writing a post so that others might absorb some of my newfound knowledge. Yikes! At this rate, in 19 weeks i'm going to be well on my way to coding genius!

Ok, let's get to it. Contrary to popular novice belief, Git and GitHub are not the same thing.

Big difference #1: Git lives locally on your computer, while GitHub lives online. Git is a software that you download and use in the command line of your computer. You can access GitHub from anywhere with an Internet connection. The name really is quite fitting: GitHub is a remote Hub for what you've done with Git on your computer.

Big difference #2: Git is all you; GitHub is all about collaboration. You have your Git and I have mine--remember it's just a piece of software on your computer. But GitHub is like a huge remote warehouse that holds all your stuff (code). Others (like say, your teammates) can access your stored repositories, enabling you to work on projects together with ease.

Big difference #3: Git and GitHub have different functionality, but they work together. Git is a version control system (VCS). This means you can use Git to save multiple versions of your code as you go along. You save your work by adding and committing changes to your code, and each time you end up with a different version. Down the line, if your code gets messy, you can return to a previous version where all was well. It's like a do-over for your code! GitHub then keeps track of all those changes for you, and makes them available online for others to view and give feedback on.

Not too complicated, right?

Embarrassing fact time: when I found out I'd be using GitHub at DBC, I gave it a quick google and still didn't quite grasp it. Not to mention Git, which I had no clue was a separate entity altogether. Hopefully this blog simplified some concepts, but the best way to learn is to go play around with this stuff yourself! Go ahead--Git in there!
