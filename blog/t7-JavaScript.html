Ruby vs JavaScript

who iterated better?

august 23, 2015


My first impressions of JavaScript are mixed. Ruby spoiled me with many shortcuts. She carries out many processes "under the hood," and doesn't need a lot of step-by-step instruction. JavaScript is a much more literal language. You must spell out exactly what you want from it.

This can be illustrated in the simple case of iterating over an array. Ruby has numerous methods for iteration, but the most basic and commonly used is each. each calls a given block of code once for each element in the array. The code below will print the double of each number in the array (on separate lines, thanks to puts).

a = [1, 2, 3, 4, 5]
a.each {|i| puts i*2}

JavaScript doesn't have as many looping functions as Ruby has looping methods. This JavaScript for loop has the same result as Ruby's each method above:

a = [1, 2, 3, 4, 5];
for (var i=0; i < a.length; i++) {
console.log(a[i]*2);
}

Comparing the two, Ruby looks more like English, while JavaScript looks like what flashes by the screen when a hacker is mad-coding in the movies. Using a for loop to iterate means defining 3 things between the parentheses:
1. Defining a counting variable and giving it a value
2. Similar to a while statement, this section defines the condition of the loop. In this case, looping will continue while our counting variable is less than the number of values in the array.
3. Incrementing our counting variable. This is done after each loop has been executed.
After the parentheses, whatever is between the curly brackets is the code block that will be executed upon each loop, same as in Ruby.

Ruby didn't need a counting variable or to be told how many loops to make. Basically everything that's between the parentheses in JavaScript's for loop is what Ruby is doing for you implicitly in the each method.

tl;dr

Ruby makes iterating easy with a lot of built-in methods that automate processes for you, while JavaScript likes to explicitly be told what to do. For me, this means Ruby is more convenient, but JavaScript makes sure I fully understand what each and every piece of my code is doing. Ding, ding! I'm declaring this round a tie!
