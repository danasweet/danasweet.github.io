outfox the box

(a guide to placing elements where you damn well please)

july 18, 2015


Making your website look like that beautiful wireframe you dreamed up can be a hearty challenge for a CSS newb. The fact is, you need to have an understanding of many properties before you can direct your elements like a theater producer ("places, everyone!"). One such property is the CSS Box Model.

Imagine that every element in your HTML file is surrounded by a shield. The shield has different layers, and its purpose is to prevent other elements from coming too close (hey, that analogy almost worked).

I'm going to disregard analogies and show a visual instead:

The 'Content' box that you see in the middle of the diagram is the bulk of your element. It could be the photo in an [img] tag, the text in a [p] tag--doesn't matter. Every element is a box, and each box has three layers that surround the content: padding, border, and margin.

We're probably all familiar with borders. Common places you might spot a border: a table data cell, a coupon in the newspaper, your best friend's instagram (not yours--borders are so 2013). So a border is a line of some sort (solid, dashed, dotted) that surrounds your content.

Padding is the space between your content and the border. It's empty space, but it inherits the properties of the element. So if the element has a background color of black, the padding will appear black as well.

In the images above, I've changed the padding from 10px to 50px. See how the space between the words (the content of the table data cell) and the cell border has enlarged? Remember, padding is the space between the content and the border.

Here's another instance of padding at work:

In the above example, I'm experimenting with my site's navigation bar. At first, I had too much padding between links. Then, too little. Sometimes it takes some trial-and-error to figure out the ideal settings for padding and margin. But as you can see in the third navigation bar, it pays off in the end!

So padding is interior, margin must be exterior, right? Indeed, margin is the space between the border and the outer edge of your element. It is transparent and will appear as white space.

Padding, border, and margin can all be set with CSS. You can define each side (top, bottom, left, right) individually, or you can make all sides equal by defining just one value, like so: "margin: 10px."

Manipulating the padding and margin of an element allows you to adjust your element's appearance and placement on the page. If you want to shift an element to the right, try increasing the margin-left. If you'd like more space above an element, try adjusting the padding-top.

Chrome Dev Tools gives us a handy visual by depicting what properties are currently affecting our element. Here, I've selected my first td (table data) cell. The content (the word 'Blue') is shown in the corner in blue. The green box surrounding it shows us that we have 50px of padding on each side. We can also see the cell border of 1px on each side. And this element has no margin set, or we would see an orange box surrounding the yellow border box.


If you found this topic less thrilling than the latest John Grishham novel, that's understandable. But when you're building your site and you finally set your margin and padding correctly and your element pops into place, well that's the stuff of dreams. Good luck outfoxing that box!
