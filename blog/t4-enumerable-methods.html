all over the .map

exploring enumerables

july 31, 2015


Ruby has so many built-in methods, it's equal parts awesome and overwhelming. Chances are, if you want to accomplish something relatively simple, there's an existing method that'll do it for you. Which is great, but they won't do you any good if you aren't familiar with them. Lately I've been diving into Ruby Docs trying to expose myself to as many methods as possible, and some of my favorites thus far are Enumberables.

To understand Enumberables, let's check out the root word. Google defines enumerate: "to mention a number of things one by one." Whoa, that makes sense (cue light bulb emoji )! Enumerables are methods that perform actions on each element one at a time--similar to the .each method that we've been using with Arrays.

Cool. Now let's look at a specific Enumerable method and how we might use it. I haven't actually used .map in my code yet, but it seems potentially helpful and versatile. (As a side note, .map has an alias method, .collect. They are essentially the same, just different name.)

.map is pretty straightforward. Call it on an object, along with a block (designated by the curly brackets) which tells it what to do to each element. The output is a new array containing the newly changed elements. .map is non-destructive by default, so if you want to modify the original array, use the destructive form of the method, .map!

["barack", "george", "bill", "george", "ronald"].map { |pres| pres.capitalize }
=> ["Barack", "George", "Bill", "George", "Ronald"]

.map comes in handy any time we need to make changes to each element in an array. In the above example, we used it to capitalize each name. Notice the code block within curly brackets defines the objects, followed by what is to be done to each element.

[2, 4, 6, 8].map! { |x| x**2 }
=> [4, 16, 36, 64]

This returns the original array with altered contents--not a new array--since we used the destructive (!) form. We're basically saying, "For each object, 'x', return the square of x."

Easy, right? I hope this has empowered you all to become method masters and brave those Ruby Docs for yourself!
